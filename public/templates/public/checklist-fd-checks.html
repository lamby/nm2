Things to check in the mailbox

  • The mailbox should be complete with the mails from all involved sides.
  • All signatures should validate.
  • SC + DMUP agreement is present.
  • Agreement SC + DMUP is signed with the key that is to go in the DD keyring: {{person.fpr|fingerprint}}.
  • No mails appear to be missing (missing bits of a thread, missing T&S without a reason...)
  • Ensure that the login name requested in the mail conversation matches the one in the AM report.
  • Ensure that the forward email address requested in the mail conversation matches the one in the AM report.

Things to check in the OpenPGP key

  • Run keycheck.sh again on the applicant key that will go in the keyring, as sometimes the AM might run keycheck on the 1024 key and forget to run in on the 4096 key.
  • Make sure key is 4096 bits, if not check if there is a good reason. Same for number of signatures, if there is only one, no other trust paths, and no good reason.
  • Check if applicant is in the DM keyring: if they are, keyring-maint prefers to move the existing key from DM to DD, so the DD key should match the DM key and the agreement should be signed with that key.
